Traditionally:
    Dev builds → QA tests later ❌

Shift-left:
    Testing starts alongside or even before development ✅

Testing on feature branch env is not shift left testing since its done after the code is written

Ideal shift left testing means
- Validating requirements before coding
- Writing test scenarios during grooming

How it looks in practice
- Writing test scenarios during requirement phase
- Reviewing user stories and acceptance criteria early
- Adding unit tests and API tests during development
